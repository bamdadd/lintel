# Web Research - python-api

## Research Scope

**Tech Area:** python-api
**Task Context:** Build a React SPA web dashboard consuming a FastAPI backend with 52 endpoints
**Research Date:** 2026-03-06

---

## Executive Summary

FastAPI has strong first-party support for all aspects of serving a React dashboard. It auto-generates OpenAPI 3.1 specs, has native SSE streaming (since 0.135.0) with keep-alive and proxy-friendly headers built in, and can serve a React SPA build via `StaticFiles` plus a catch-all route. For 52 endpoints, **Orval** in `react-query` + `tags-split` mode is the recommended TypeScript client generator -- it produces one file of typed TanStack Query hooks per endpoint tag with zero manual wiring. Single-container multi-stage Docker deployment is the right starting point.

## Findings by Topic

### 1. Serving React SPA Static Files from FastAPI (Production)

**Confidence:** 0.92

- Mount React's `dist/` with `StaticFiles`, add catch-all returning `index.html` for non-API paths
- Register API routers before the catch-all to avoid shadowing
- A custom `SPAStaticFiles` subclass (overriding `lookup_path`) is cleaner than a bare catch-all

**Sources:** [Serving React with FastAPI](https://davidmuraya.com/blog/serving-a-react-frontend-application-with-fastapi/) | [FastAPI Static Files docs](https://fastapi.tiangolo.com/tutorial/static-files/)

### 2. FastAPI CORS Configuration for React Dev Server

**Confidence:** 0.95

- `CORSMiddleware` listing `http://localhost:5173` explicitly
- Gate on `ENV=development` so absent in production
- `allow_origins=["*"]` is incompatible with `allow_credentials=True`
- Native `EventSource` API does not support custom headers; use `@microsoft/fetch-event-source`

**Sources:** [FastAPI CORS docs](https://fastapi.tiangolo.com/tutorial/cors/) | [TestDriven FastAPI+React](https://testdriven.io/blog/fastapi-react/)

### 3. OpenAPI Spec Generation and TypeScript Client Tools

**Confidence:** 0.92

- Set `generate_unique_id_function` for clean operation IDs (become TS function names)
- **Orval** with `react-query` + `tags-split` mode generates typed TanStack Query hooks per tag
- **@hey-api/openapi-ts** is the alternative for a minimal fetch client
- Offline spec export: `app.openapi()` without starting server

**Sources:** [FastAPI Generate Clients](https://fastapi.tiangolo.com/advanced/generate-clients/) | [Orval](https://orval.dev/)

### 4. SSE for Real-Time Event Streaming

**Confidence:** 0.93

- FastAPI 0.135.0 added native SSE: `from fastapi.sse import EventSourceResponse, ServerSentEvent`
- Built-in keep-alive pings (15s), `Cache-Control: no-cache`, `X-Accel-Buffering: no`
- React client: `@microsoft/fetch-event-source` (supports POST and Authorization headers)
- SSE vs WebSocket: SSE correct for Lintel -- server pushes events, browser only reads
- HTTP/1.1 limit: 6 SSE connections per domain; use single stream with filtering or HTTP/2

**Sources:** [FastAPI SSE docs](https://fastapi.tiangolo.com/tutorial/server-sent-events/) | [SSE with FastAPI and React](https://www.softgrade.org/sse-with-fastapi-react-langgraph/)

### 5. Deployment Patterns

**Confidence:** 0.88

**Pattern A -- Single Container (recommended):**
Multi-stage Dockerfile: Node stage builds React, Python stage copies `dist/` and runs uvicorn.

**Pattern B -- NGINX + Two Containers:**
NGINX serves React, proxies `/api/` to FastAPI. Better for separate CI pipelines and CDN.

**Sources:** [Single container guide](https://dakdeniz.medium.com/fastapi-react-dockerize-in-single-container-e546e80b4e4d)

### 6. API Versioning

**Confidence:** 0.85

- `APIRouter(prefix="/api/v1")` with plain router prefixes is sufficient
- SPA catch-all covers all non-`/api/` paths cleanly
- Adopt versioning libraries only when maintaining two simultaneous API versions

### 7. CI/CD Workflow for TypeScript Client

**Confidence:** 0.88

- Commit `openapi.json` to repo, generated by `app.openapi()` without starting server
- Run Orval in pre-commit hook: `uv run python scripts/export_openapi.py && bunx orval`
- End-to-end type safety: `Pydantic -> OpenAPI 3.1 -> Orval -> typed React hooks`

**Sources:** [PropelAuth CI autogen](https://www.propelauth.com/post/autogenerating-clients-with-fastapi-and-github-actions)

## Best Practices Summary

### Strongly Recommended (0.85+ confidence)
1. `generate_unique_id_function` on FastAPI init for clean TypeScript function names
2. Orval `react-query` + `tags-split` mode for 52 endpoints
3. Native FastAPI SSE for event streaming
4. `CORSMiddleware` gated on `ENV=development`
5. Catch-all `/{full_path:path}` registered last for SPA fallback

### Consider (0.7-0.85 confidence)
6. Commit `openapi.json` to repo for reproducible client generation
7. Multi-stage single-container Docker build
8. `APIRouter(prefix="/api/v1")` for versioning (no library needed)
